Verification leg & scaffold removal zwo · c23 / pjj / xtp

The methodology mentioned the v3.0 verification scripts nowhere, so the model would never run them. The leg is now documented across methodology.md, report-assembly.md, quality-gates.md and SKILL.md, with the load-bearing ordering pinned (see the pipeline diagram).

research_engine.py — 584 lines of self-admitted scaffold that printed instruction strings and never searched — was deleted after confirming zero external imports (via LSP findReferences). That mooted the --resume bug (pjj) and the Optional-access fixes (xtp) outright.

Bug fixes k4z · 0gn

k4z — URL verification stopped flagging valid citations #

before 
req = request.Request(url, method='HEAD')
req.add_header('User-Agent',
  'Mozilla/5.0 (Research Citation Verifier)')
with urlopen(req) as r:
    if r.status == 200:
        return True, "URL accessible"
    else:                  # unreachable
        return False, ...
after 
# browser UA + HEAD->GET fallback on
# method/client-reject codes; 2xx/3xx ok
def _attempt(method):
    req = Request(url, method=method,
        headers=_BROWSER_HEADERS)
    with urlopen(req) as r:
        if 200 <= r.status < 400:
            return True, ...

The old code custom-UA'd a HEAD request that Cloudflare and friends 403, and its non-200 branch was unreachable because urlopen raises on non-2xx. Now valid citations behind bot-protection resolve.

0gn — md_to_html actually writes a file now #

main() hand-rolled sys.argv (so --help crashed) and only printed a 1000-char preview. It now has real argparse (-o/--out, --template, --fragments-only), substitutes the report into the McKinsey template, and writes a complete HTML document to disk.

Gates & correctness rp4 · r7t · loa · aq0 · 3qs

rp4 — source floor is now a tiered gate

It previously only ever warned. It now errors (non-zero exit) below 5 sources — the documented hard stop — while keeping 5–9 as a warning so the documented graceful-degradation band isn't broken.

r7t — the claim-support gate discriminates weak support

before 
# only zero-link claims ever failed
if args.strict and factual_unsupported > 0:
    passed = False
after 
# weak/irrelevant support fails too
factual_failing = (factual_unsupported
                   + factual_needs_review)
if args.strict and factual_failing > 0:
    passed = False

Before, any factual claim linked to any evidence row passed regardless of relevance. Now needs_review (overlap score < 0.35) fails --strict too.

loa — one numbering authority, no drift

Three sites derived [N] independently (one counted duplicates, one deduped). They now all call a single build_display_map(sources) helper, verified consistent even when sources.jsonl contains a duplicate source_id.

aq0 — source_evaluator got a CLI; 3qs — schemas demoted to docs

source_evaluator.py was orphaned (referenced by the methodology but only had a demo block); it now has a real --json CLI. The JSON Schemas in schemas/ are explicitly documentation-only (a stdlib-only project adds no jsonschema dependency), and claim.schema.json now documents the internal _-prefixed fields it previously forbade.
