What this is

A Claude Code skill — a bundle of SKILL.md + reference docs + scripts that teaches the model a disciplined, citation-first research workflow.

The Python in scripts/ is deliberately not an orchestrator. The model does the fuzzy work (decomposing the question, searching, reading, synthesizing); the scripts do the deterministic work (assigning stable IDs, storing evidence, scoring claim support, validating structure). This split is the whole point of the v3.0 design: the parts a language model is unreliable at — never dropping a citation, never letting a claim drift from its evidence — are handled by code and persisted to disk, so they survive context compaction and continuation.

The v3.0 data model Append-only JSONL with stable sha256(...)[:16] IDs, tied together by a run manifest.

Source identity is a hash of the canonical locator (DOI > arXiv > normalized URL), so it is stable across edits, renumbering, and continuation. Display numbers [N] are derived at render time and never stored.

The verification leg Runs after the report is drafted. Ordering is load-bearing — see the callout below.

Why the order matters
[N] is a source's position in sources.jsonl. If you draft [5] and only then register a different 5th source, the citation silently corrupts — and because the store is append-only, that is detectable but not fixable without re-drafting. So: register every source before drafting, run assign-display-numbers, then draft using exactly that map.

The gate itself is deterministic: it resolves each claim's [N] back to a source_id, gathers the linked evidence quotes, and scores support via entity / number / date / lexical-overlap checks — no model call. Under --strict it exits non-zero if any factual claim is unsupported or needs_review.
