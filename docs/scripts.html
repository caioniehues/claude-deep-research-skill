citation_manager.py evidence pipeline

Owns source identity and the run manifest. source_id = sha256(canonical_locator)[:16], where the canonical locator prefers DOI, then arXiv, then a normalized URL (lowercased host, fragment and tracking params stripped).

It is also the single home of build_display_map(sources) — the one authority for [N] numbering, shared by assign-display-numbers, export-bibliography, and the verifier.

# initialise a run (creates manifest + empty JSONL files)
python scripts/citation_manager.py init-run --out-dir [dir] --query "..." --mode deep

# register a source -> returns a stable source_id (re-registering is a no-op)
python scripts/citation_manager.py register-source --json '{"raw_url":"...","title":"..."}' --dir [dir]

# after ALL sources are registered, emit the source_id -> [N] map
python scripts/citation_manager.py assign-display-numbers --dir [dir]

# render the bibliography (markdown or json)
python scripts/citation_manager.py export-bibliography --dir [dir] --style markdown

evidence_store.py evidence pipeline

Appends evidence spans pulled from sources. evidence_id = sha256(source_id + normalized_quote + locator)[:16], so the same quote from the same source dedupes. Evidence must be persisted here before synthesis so continuation agents and the claim-support gate can see the full trail.

# add an evidence span tied to a source
python scripts/evidence_store.py add --json '{"source_id":"...","quote":"exact text",
  "evidence_type":"direct_quote","locator":"page 5"}' --dir [dir]

python scripts/evidence_store.py list --dir [dir] --source-id [id]

extract_claims.py evidence pipeline

Parses the drafted report into atomic, typed claims. Each sentence becomes a claim with a claim_type — factual (hard-fails on no support), synthesis, recommendation, or speculation (no support gate).

Contract hole worth knowing
extract captures each sentence's [N] into _citation_numbers but leaves cited_source_ids empty (linking is deferred to the verifier). The manual add subcommand does not write _citation_numbers, so any claim added that way must pass cited_source_ids explicitly or it will never link. 
python scripts/extract_claims.py extract --report [report.md] --dir [dir]
python scripts/extract_claims.py stats --dir [dir]

verify_claim_support.py evidence pipeline gate

The deterministic, no-LLM claim-support gate. It resolves each claim's [N] back to a source_id via build_display_map, gathers linked evidence quotes, and scores support with a weighted blend of token overlap, number match, year match, and entity match.

  • score ≥ 0.6 → supported
  • 0.35 ≤ score < 0.6 → partial
  • score < 0.35 → needs_review
# --strict exits non-zero on any factual claim that is unsupported OR needs_review
python scripts/verify_claim_support.py verify --dir [dir] --strict
python scripts/verify_claim_support.py report --dir [dir]

source_evaluator.py scoring

Scores a source 0–100 across domain authority, recency, expertise, and bias, and returns a trust recommendation.

python scripts/source_evaluator.py --json '{"url":"https://nature.com/...",
  "title":"...","publication_date":"2025-10-15","author":"..."}'

# -> {"overall_score": 86.5, ..., "recommendation": "high_trust"}

validate_report.py quality gate

Nine structural checks: executive-summary length, required sections, citation formatting, bibliography integrity, placeholder text, content-truncation, word count, the source floor, and broken internal links.

Source floor is a real gate
Fewer than 5 sources is a hard error (non-zero exit); 5–9 is a warning (note in Limitations); 10+ passes clean. 
python scripts/validate_report.py --report [report.md]
# exit 0 = pass, exit 1 = errors present

verify_citations.py quality gate

Catches fabricated citations: resolves DOIs, checks URL reachability, and runs hallucination heuristics (future years, anachronistic AI terms in old papers, templated titles, recent claims with no verification path).

Reachability that doesn't false-positive
URL checks send a real browser User-Agent and fall back from HEAD to GET when a server rejects the method or client, treating any 2xx/3xx as reachable — so valid citations behind bot-protection are no longer flagged as dead. 
python scripts/verify_citations.py --report [report.md] [--strict]

md_to_html.py rendering

Converts a markdown report into a full, self-contained HTML document by substituting the report into the McKinsey-style template ({{CONTENT}}, {{BIBLIOGRAPHY}}, {{TITLE}}, {{DATE}}, {{SOURCE_COUNT}}). Falls back to a minimal HTML5 skeleton if no template is found.

python scripts/md_to_html.py [report.md]                # writes report.html next to it
python scripts/md_to_html.py [report.md] -o out.html
python scripts/md_to_html.py [report.md] --fragments-only  # just the HTML fragments

verify_html.py rendering

Confirms the rendered HTML faithfully mirrors the markdown: all sections present, no unreplaced {{placeholders}}, no emojis, valid document structure, and citations / bibliography carried across.

python scripts/verify_html.py --html [report.html] --md [report.md]

Testing & types

The suite is stdlib unittest only (no pytest), invoking the CLIs via subprocess and importing pure functions where network calls must be avoided.

python -m unittest discover tests   # 80 tests
npx pyright                         # 0 errors / 0 warnings (see pyrightconfig.json)
